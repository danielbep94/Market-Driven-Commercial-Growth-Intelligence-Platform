NAMING CONVENTIONS
================================================================================

Layout files (.txt):
  Format  : {SOURCE_KEY}.txt  or descriptive name (e.g., EDP_MARKET.txt)
  Content : JSON-like dict with keys:
    sql_ref       в†’ source SQL reference
    grain_hint    в†’ human-readable grain description
    business_keys в†’ non-date dimension columns
    grain_cols    в†’ all grain columns (date + dims)
    metric_cols   в†’ aggregated measure columns
    date_col      в†’ primary date column
    source_key    в†’ registry key (must match configs/semantic_registry.yaml)

Infrastructure files (.txt):
  SNOWFLAKE_CONNECTION.txt  в†ђ connection profiles + validation summary
  CONNECTION_ISSUES.txt     в†ђ SQL error catalogue + troubleshooting guide
  FOLDER_STRUCTURE.txt      в†ђ this file

Config files (.yaml):
  Prefix  : descriptive name, lowercase with underscores
  Registry: configs/semantic_registry.yaml (auto-generated, do not edit manually)
