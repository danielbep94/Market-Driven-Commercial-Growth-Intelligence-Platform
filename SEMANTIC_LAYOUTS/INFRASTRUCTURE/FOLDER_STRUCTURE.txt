================================================================================
NAMING CONVENTIONS
================================================================================

Layout files (.txt):
  Format  : {SOURCE_KEY}.txt  or descriptive name (e.g., EDP_MARKET.txt)
  Content : JSON-like dict with keys:
    sql_ref       → source SQL reference
    grain_hint    → human-readable grain description
    business_keys → non-date dimension columns
    grain_cols    → all grain columns (date + dims)
    metric_cols   → aggregated measure columns
    date_col      → primary date column
    source_key    → registry key (must match configs/semantic_registry.yaml)

Infrastructure files (.txt):
  SNOWFLAKE_CONNECTION.txt  ← connection profiles + validation summary
  CONNECTION_ISSUES.txt     ← SQL error catalogue + troubleshooting guide
  FOLDER_STRUCTURE.txt      ← this file

Config files (.yaml):
  Prefix  : descriptive name, lowercase with underscores
  Registry: configs/semantic_registry.yaml (auto-generated, do not edit manually)
